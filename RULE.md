# Documentation Rules

1. **Keep `README.md` clean**: The root `README.md` should only act as a table of contents and quick start guide. Do not paste long blocks of code or architecture decisions there.
2. **Update `execute.md`**: Whenever a phase is completed, make sure to check off the related tasks in `execute.md`.
3. **Docs sync**: If the database schema or core architecture changes, be sure to update `docs/architech.md`.
4. **Link paths**: Always use relative paths (e.g., `./docs/planning.md`) when linking to internal documents.
